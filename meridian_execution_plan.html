Person 1 — AI / agents lead
P1

AI / agents lead

schemas.py · graph.py · planner.py · interaction_analyzer.py · knowledge_retriever.py · risk_assessor.py · nba_generator.py

▸Day 1 Foundation: schemas + graph + first 2 agents
→Define all Pydantic schemas in backend/models/schemas.py: AgentStep(agent_name, action, reasoning, duration_ms), Evidence(id, source_type, source_name, excerpt, relevance_score), Action(title, description, confidence, estimated_impact, evidence_ids, precedent_accounts), MemoryContext(similar_cases, confidence_boost, base_confidence, boosted_confidence), RecommendationOutput — all other agents produce from these
→Set up LangGraph StateGraph in graph.py. State dict keys: raw_input, signals, knowledge_chunks, risk, recommendations, agent_trace. Each agent reads state, appends to agent_trace, writes its output key
→Implement planner.py: reads raw_input, always invokes all 4 specialist agents in order, logs AgentStep with a brief reasoning string and start/end timestamps. Returns execution_plan: list[str]
→Implement interaction_analyzer.py: prompt Claude with the meeting transcript, return signals: list[dict] each with text, type (risk|positive|neutral), severity (high|medium|low). Log AgentStep
▸Day 2 Remaining agents + confidence scoring
→Implement knowledge_retriever.py: import vector_store.py from P2, call search(query, collection="kb", n=5) using signal texts as queries. Return top-5 Evidence objects with relevance_score. Log AgentStep
→Implement risk_assessor.py: prompt Claude with signals + retrieved knowledge chunks. Output structured JSON: risk_score (0.0–1.0), risk_level (critical|high|medium|low), signal_count, risk_factors: list[str]. Map each factor to an evidence_id. Log AgentStep
→Implement nba_generator.py: receives risk assessment + knowledge + episodic memory context from P2. Prompt Claude to output: 1 primary action + 2 alternatives, each with title, description, confidence (0.0–1.0), estimated_impact, evidence_ids, precedent_accounts. Apply memory confidence boost: boosted_confidence = min(base * (1 + 0.06 * similar_count), 0.97)
→Wire all agents in graph.py and expose run(raw_input: str, account_id: str) → RecommendationOutput function that P3's FastAPI calls
▸Day 3 Prompt tuning + integration
→Tune all agent prompts against the 3 demo scenarios. Target: risk scores within ±5% of expected, primary recommendation matches expected action
→Add duration_ms to every AgentStep using time.time() before and after each agent call
→Run tests/eval_scenarios.py and fix any agent producing wrong risk level or irrelevant NBA
→Help P3 with architecture diagram: explain the LangGraph flow + state transitions

Data / memory lead

data/* · vector_store.py · episodic.py · scripts/ingest.py · scripts/seed_memory.py

▸Day 1 All content + ChromaDB ingestion pipeline
→Write 10 meeting transcripts in data/meeting_notes/. Use real dialogue format (speaker labels, specific numbers, product feature names). 3 at-risk (Acme Corp, BlueWave Tech, Nexus Systems), 3 healthy/expansion (Globex Corp, Umbrella Corp, Initech), 2 onboarding (TechCorp, Quantum Labs), 2 to-be-resolved (GlobexQ1, InitechQ4) — these last two seed memory
→Write 15 knowledge articles in data/playbooks/ as .md files: churn_prevention.md, ebr_guide.md, onboarding_best_practices.md, expansion_playbook.md, escalation_procedures.md, health_score_methodology.md, qbr_template.md, champion_change.md, renewal_risk_signals.md, product_reporting_module.md, product_integrations.md, product_automation.md, competitive_positioning.md, success_plan_template.md, sla_policy.md
→Write 5 customer profiles in data/customer_profiles/ as .json: fields = account_name, arr, industry, contract_end, csm, health_score_history: [float], daily_active_users, last_login_date, features_adopted: [str], open_tickets, nps_score
→Implement backend/memory/vector_store.py: ChromaDB client with 2 named collections — "knowledge_base" and "resolved_cases". Expose: add(texts, metadatas, collection), search(query, collection, n=5) → list[dict] using sentence-transformers/all-MiniLM-L6-v2
→Write scripts/ingest.py: reads all files in data/playbooks/ + data/meeting_notes/, splits into 500-token chunks, embeds and stores in "knowledge_base". Metadata per chunk: source_file, chunk_index, category. Run it and confirm: collection.count() >= 80
▸Day 2 Resolved cases + episodic memory loop
→Write 6 resolved cases in data/resolved_cases/ as .json: fields = case_id, account_name, arr, situation_summary, risk_signals: [str], action_taken, outcome, days_to_resolution, renewal_achieved: bool, csm_notes. 4 should be EBR-resolved, 1 re-onboarding, 1 champion change. These are what the memory system learns from
→Implement backend/memory/episodic.py: SQLite table memory_log(id, account, risk_score, risk_level, recommendation_title, decision, modification_notes, outcome, timestamp). Expose: log_feedback(data) and find_similar(risk_level, risk_score, top_n=3) → list[dict] using risk_level + risk_score proximity as similarity
→Write scripts/seed_memory.py: reads all data/resolved_cases/ files and inserts them into episodic.py memory_log as pre-existing resolved cases. Run this before the Day 3 demo to ensure memory has data
→Expose get_memory_context(risk_level, risk_score) → MemoryContext in episodic.py that P1's nba_generator.py calls. Returns: similar_cases_found, precedent_accounts: [str], base_confidence, boosted_confidence
▸Day 3 Demo prep + eval scenarios
→Run seed_memory.py to pre-load all resolved cases. Verify: SELECT count(*) FROM memory_log returns ≥ 6
→Write tests/eval_scenarios.py: 5 test cases each with input_transcript, expected_risk_level, expected_primary_action_keyword. Assert outputs match. P3 runs this to confirm system works before recording
→Prepare demo state files in docs/demo_script.md: which account to load for each scenario, what to say, when to click Modify, what feedback to type
Person 3 — Frontend / demo lead
P3

Frontend / demo lead

backend/main.py · frontend/app.py · frontend/components/* · docs/* · README.md

▸Day 1 FastAPI skeleton + Streamlit shell + agent trace UI
→Build backend/main.py FastAPI app. POST /analyze: accepts {"account_id": str, "interaction_text": str}, calls graph.run(), returns RecommendationOutput. POST /feedback: accepts {"request_id", "decision", "modification_notes"}, calls episodic.log_feedback(). GET /health: returns {"status": "ok"}
→Build frontend/app.py Streamlit skeleton: left sidebar with account dropdown (5 accounts), paste-transcript textarea, and "Analyse account" button. Main area starts empty with a centered placeholder
→Build frontend/components/agent_trace.py: render_agent_trace(steps: list[AgentStep]) renders an st.expander("Agent trace", expanded=True) with one row per step showing the agent name badge, action text, reasoning in muted text, and duration in ms right-aligned
▸Day 2 Full recommendation panel + HITL modify path
→Build frontend/components/recommendation_panel.py: risk score row (big number + colored bar + signal count badge), then 3 evidence cards each with source-type badge (Meeting note / Playbook / Memory), excerpt, and relevance bar
→Build frontend/components/memory_panel.py: only renders when memory_context.similar_cases_found > 0. Shows: "Memory lookup: N similar cases found", account names as badges, and a confidence boost bar showing base_confidence → boosted_confidence with the delta highlighted in green
→Build frontend/components/hitl_widget.py: renders 3 buttons. Accept → calls /feedback with decision="accept", shows success toast. Reject → same with decision="reject". Modify → shows a st.text_area with placeholder "What would you like to change?" and a Submit button that calls /feedback with decision="modify" + notes, then re-runs /analyze with the feedback appended to the original prompt. Display "Updated based on your feedback: [what changed]"
→Wire everything in app.py: on button click, call /analyze via httpx, store result in st.session_state, render trace → risk → evidence → memory → primary recommendation → alternatives → HITL
▸Day 3 Architecture + README + video
→Draw architecture diagram in docs/ARCHITECTURE.md: show the 5-agent LangGraph flow, ChromaDB (2 collections), SQLite memory, FastAPI, and Streamlit. Use Mermaid syntax — renders natively on GitHub
→Write README.md: one-line product description, prerequisites, pip install -r requirements.txt, python scripts/ingest.py, python scripts/seed_memory.py, uvicorn backend.main:app, streamlit run frontend/app.py. Include a screenshot of the recommendation panel
→Record 5-min demo video following the 3-scenario arc: Acme Corp (at-risk, no memory) → accept recommendation → TechCorp (same type, memory kicks in, confidence boosted) → show architecture diagram. Screen-record with a talking head overlay if possible
→Record 5-min architecture walkthrough: walk through each agent in graph.py, explain LangGraph state transitions, show ChromaDB collections, explain memory boost formula, show Pydantic schemas
→Final push: tag release as v1.0.0-hackathon, verify README renders correctly on GitHub, share repo link

Pydantic schema — the contract between all components
class AgentStep(BaseModel): agent_name: str; action: str; reasoning: str; duration_ms: int class Evidence(BaseModel): id: str; source_type: Literal["meeting_note","playbook","memory","crm"] source_name: str; excerpt: str; relevance_score: float class Action(BaseModel): title: str; description: str; confidence: float estimated_impact: str; evidence_ids: List[str]; precedent_accounts: List[str] class MemoryContext(BaseModel): similar_cases_found: int; confidence_boost: float base_confidence: float; boosted_confidence: float; precedent_accounts: List[str] class RecommendationOutput(BaseModel): request_id: str; account_name: str; csm_name: str; renewal_date: str risk_score: float; risk_level: Literal["critical","high","medium","low"]; signal_count: int agent_trace: List[AgentStep]; evidence: List[Evidence] primary_recommendation: Action; alternatives: List[Action] memory_context: Optional[MemoryContext]; generated_at: datetime
